(These instructions are partially based on a text file written by Reg, which was included in the FrobTADS download. None of the creators of this site have access to a Mac, so a lot of research was done to write this, without any direct testing or experience.)
  1. Open Finder, and navigate to Applications.
  2. Open the Utilities folder.
  3. Run Terminal.app, and also drag it to the dock for easier access in the future.
  4. For those unaccustomed to the Terminal, Reg provides the following message:

    Using terminal, you are always located somewhere inside your computer. For instance, you could be inside your applications folder, or perhaps your itunes folder... but it's important to know where you are because this affects how commands could be interpreted.

    Some common commands:
    CHANGING DIRECTORY
    • $cd
      Change directory (This is how you navigate around in your computer).
    • $/Applications
      The slash refers to a folder, the name after the slash is the name of a folder.
    • $cd /Applications
      This is how you would enter the Applications folder.
    • $cd /Applications/Utilities
      This is how you would enter the utilities folder.
    • $.
      A dot means "current folder".
    • $..
      Two dots mean "one folder before".
    • $cd ../
      Means move to the previous folder.

    Try the cd command with different folders to get a feel for navigating.

    If any folders have spaces in them, things are a little more complicated. You can drag a file or folder into the "terminal" window and it will show how you would type the location of the file or folder.

    FINDING A FILE ( Includes hidden files which spotlight won't show you )
    • $cd /
      Changes directories so you're located in the top-most directory.
    • $find . -name adv3.h
      Searches for a file called "adv3.h".
    OTHER COMMANDS
    • $tar -xzf
      Expand a *.tar.gz compressed archive.
    • $ls
      List all files in a folder, except hidden files.
    • $ls -a
      List everything in a folder, including hidden files.
    • $sudo
      This command is added to the beginning of a command if it's something where you need to have administrative (root) access.

    A list of more commands
    A very good unix beginner page is here

    By the way:
    The word binary generally means compiled file as opposed to the source code which is the building blocks of a file.

  5. Install Xcode Tools.

    Reg says:

    This is on your install DVD that came with your mac but it's better to download the newest version via the internet.

    This is free when you make an ADC [or Apple Developer] account (account is also free).

    Follow the installation instructions that come with it.

  6. Head to the download site for Homebrew, and download the pkg installer.
  7. Double-click on the Homebrew pkg file to install it.
  8. If you have Terminal open, close it and open a new one. Otherwise, open Terminal.app now.
  9. Enter this command: $echo $PATH
  10. What will appear is a list of folder paths, where each step in the path is separated by a / (slash), and each path is separated by a : (colon). Search through the paths, and make sure you find one that says /usr/local/bin.

    If you do find this, skip to step 15.
  11. Open your home folder in Finder.
  12. Look for a file called .bashrc. If you do not see it in your home folder, press SHIFT + CMD + ..

    Open .bashrc with TextEdit.

    Make sure TextEdit is using plain text by pressing SHIFT + CMD + T!
  13. Scroll to the bottom of the file, and add the following line: {% highlight bash %} export PATH="/usr/local/bin:$PATH" {% endhighlight %}
  14. Save and close the text file.
  15. Close the terminal.
  16. Open a new terminal, and run $brew install cmake.
  17. Run $brew install ncurses.
  18. Close the terminal.
  19. Download the latest FrobTADS source code.
  20. Reg says:
    Your computer will probably automatically decompress these items. If it hasn't done so for the frobtads folder, double-click on the disk image so it decompresses.
  21. Move the decompressed frobtads-2.0 folder to your home folder.
  22. Start Terminal.app, and run $cd ~/frobtads-2.0.
  23. Make sure you are actually in the folder. $ls should list filenames such as:
    • CMakeLists.txt
    • CMCommon.cmake
    • etc...
    If you only see frobtads-2.0, then try $cd frobtads-2.0 && ls to repeat this verification step.
  24. $mkdir build
  25. $cd build
  26. Simply run $cmake ..
  27. Finally, run the compile process with this command:
    $sudo cmake --build . --target install
  28. If the research and efforts of the writer of this guide have paid off, then the compilation process should be successful, and you will have a new Terminal.app command at your disposal: $t3make, which is used to compile new TADS 3 games from your game project folders.
