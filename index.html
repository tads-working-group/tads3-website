Empowering Ambitious Authors

TADS 3 is a powerful and flexible interactive fiction authoring system designed and built for ambitious authors. It brings to the table a powerful and dynamic yet fun to use object-oriented language purpose built for writing interactive fiction, two advanced standard libraries that push the boundaries of what interactive fiction is capable of both in simulation and in customizability, as well as unparalleled multimedia capabilities.

Core benefits

  • Powerful

    Want to write a story with incredible simulational sophistication, dynamic runtime capabilities, advanced programming constructs, wild new game mechanics, or something else? TADS is capable of entire categories of feats impossible for other interactive fiction authoring systems. Authors switching to TADS will find new fields of imagination opened to them!

  • Simulation-Oriented

    Interested in simulation? Both of TADS 3's standard libraries come with rich and fully-featured world models with all of the features you could possibly need for even the most complex and simulation-heavy parser game. Nevertheless, if you don't need these features, they fade into the background cleanly.

  • Flexible

    Want to use new senses, or navigate in non-Euclidian directions? Although TADS 3's standard libraries come with highly complex world models already, those assumptions are just your starting point! TADS 3 is designed to make your own code, the standard library code, and even the langauge itself, flexible and extensible in a way that makes games fun to write and easy to iterate on.

  • Power-User Friendly

    Already comfortable with programming? TADS 3 is designed with advanced interactive fiction authors in mind, offering the clarity, conciseness, and specificity of a fully-fledged programming language with all the capabilities you would expect and much more. TADS is confident in its authors' ability to use complex, powerful, and advanced feature sets, which enables it to do a lot more without worrying about whether its users can keep up.

Ready to learn more?

TADS 3 is comprised of several core components:

  1. A special-purpose object-oriented programming language designed with concise yet readable syntax and a focus on extensibility, dynamic typing, and ease of defining new objects (and sub-objects including contained and component objects) with custom properties and methods without having to declare a new class. The TADS 3 language makes growing and modifying your game — and the standard library! — fast, fun, and easy.
  2. A text-adventure-specific standard library (two, in fact) containing an advanced world model that includes things like room parts, sense-passing, a complex understanding of attachable and detachable sub components and object relationships, and hundreds of different powerful properties to customize the precise behavior of each object, an advanced and extremely customizable parser that can understand arbitrary custom sentences, and perform user-customizable spell check — and more.
  3. A performant compiler to compile those programs,
  4. and an advanced virtual machine to run TADS programs and produce output.

In addition, there are also different kinds of interpreters that use the virtual machine to run your TADS games and then interpret the output produced by the games into the multimedia output and graphical user interface players see.

To learn more about, and download, these components, see below

TADS 3 also has an extensive set of learning resources, including four introductory books, a System Manual, a Technical Manual, a Library Manual, and two Reference Manuals, all of which are many hundreds of detailed, orderly, and clear pages long. If you've struggled with the documentation of other interactive fiction authoring systems in the past, or are concerned that using such a niche piece of software will make answers hard to find, fear not! Michael J. Roberts and Eric Eve have ensured that no TADS programmer will ever stay confused for long. Getting an interpreter

There are three types of TADS interpreters: full-featured HTML TADS interpreters such as the confusingly named HTML TADS for Windows and the premier interpreter QTADS, which can handle the full multimedia capabilties of TADS, including multiple sound channels, images, text styling and centering, clickable buttons and links, and much more; text-only TADS interpreters such as FrobTADS, multi-format interpreters such as Gargoyle and Parchment, and the DOS TADS interpreter that just ignore any output that isn't text, and Web UI TADS which is where the virtual machine itself acts as a server serving up a web-based version of an HTML TADS interpreter which can actually embed arbitrary HTML interface elements and JavaScript scripts and can communicate with them via AJAX or WebSockets.

A brief history of TADS

Beginnings

TADS 1.0 was written and designed by Michael J. Roberts and released in September of 1990 as shareware under the imprint of "High Energy Software", a company created by Roberts and his friend Steve McAdams, at a time when the only other option for writing interactive fiction was AGT. It was fairly popular at the time, spawning a trilogy of shareware Zork homage/parodies that were "the most concerted and sustained effort at building a real business out of [the shareware IF scene]" at the time.1

Released as Open Source

By the time of TADS 2.2.1, released in October of 1996, due to a lack of money in the shareware business, TADS was released as freeware: although not stricty free software, since forks and modifications for the purpose of anything but porting were verboten, free use, copying, and distribution were now allowed.

TADS 3

Also open source freeware, TADS 3, first released on September 15, 2006, was designed by Michael J. Roberts as a brand new revitalization superceding TADS 2. Despite being released in the early oughts, TADS 3 still pushes the boundaries of interactive fiction authoring system capabilities and world models in many ways, including with runtime object cloning and creation, multi-channel audio capabilities, and sense-passing for any kind of sense, even user-defined ones. The core appeal of TADS goes beyond specific features, however.

If you want to learn more about the history of TADS, I recommend checking out the Digital Antiquarian's article here. Unfortunately, that article only covers the early history, and what we know of the rest is spotty at best.
