1. Assuming you already have VSCode installed, start it up.
  2. Install these extensions:
    1. vscode-tads3tools by Tomas Ã–berg, and
    2. Code Spell Checker by Street Side Software
  3. Go to File > Preferences > Settings.
  4. Enter tads into the settings search bar, and click on the Tads category, under Extensions.
  5. Verify that Tads > Preprocessor: Path and Tads3 > Compiler: Path are both set to run the t3make binary.
  6. Verify that you have the Adv3 and Adv3Lite libraries installed. If you had compiled with the /usr/local prefix, then they should be under /usr/local/share/frobtads/tads3/lib/adv3/ and /usr/local/share/frobtads/tads3/lib/adv3Lite/, respectively.
  7. Verify that Tads3: Include points to your TADS 3 include files. If you had compiled with the /usr/local prefix, then they should be under /usr/local/share/frobtads/tads3/include/.
  8. Verify that Tads3: Lib points to your TADS 3 library files. If you had compiled with the /usr/local prefix, then they should be under /usr/local/share/frobtads/tads3/lib/.
  9. Verify that Tads3 > Adv3: Library points to your TADS 3 Adv3 directory. If you had compiled with the /usr/local prefix, then they should be under /usr/local/share/frobtads/tads3/lib/adv3/.
  10. Verify that Tads3 > Adv3 Lite: Library points to your TADS 3 Adv3Lite directory. If you had compiled with the /usr/local prefix, then they should be under /usr/local/share/frobtads/tads3/lib/adv3Lite/.
  11. Verify that Tads3: Game Runner Interpreter is set to run your preferred TADS 3 interpreter. For example, if you are using QTADS to play games, you might have this set to something like {% if selected_os == "MacOS" %} /Users/username-here/Desktop/QTads/QTads.app. {% else %} /home/username-here/Desktop/QTads/QTads.AppImage. {% endif %}
  12. Close the Settings tab.
  13. Press {% if selected_os == "MacOS" %} CMD + SHIFT + P {% else %} CTRL + SHIFT + P {% endif %} to open your command search bar, and type in create a new Tads3 template project. Select the top result to run it.
  14. A dialog window will open, where you must select your game's project directory.
  15. Once the project directory is set, you will asked to choose either an Adv3 project template, or an Adv3Lite project template.
  16. A basic template will be generated in the project directory. Each time you save a file in your project, t3make will be run automatically. It will use the compile options specified in Makefile.t3m, and the game file will usually be named gameMain.t3, unless you specify otherwise in the makefile. The source file, by default, will be named gameMain.t.
  17. Open gameMain.t, if it's not open already.
