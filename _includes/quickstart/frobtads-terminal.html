1. For the rest of this guide, there will no specific instructions for your editor of choice, so it is assumed you are using something with capabilities equal to—or surpassing—the nano terminal-based text editor.

    At this point, create a new project directory for your game, and navigate into it.
  2. Decide at this time if you are using either the Adv3 or Adv3Lite library for your game.
  3. Create your first source file: gameMain.t.
    For an Adv3 template {% include quickstart/adv3/gameMain.md %} For an Adv3Lite template {% include quickstart/adv3lite/gameMain.md %}
  4. Create your project makefile: Makefile.t3m.
    For an Adv3 template {% include quickstart/adv3/makefile.md %} For an Adv3Lite template {% include quickstart/adv3lite/makefile.md %}
  5. Create a directory named obj in your game project directory. This can be done with $mkdir obj.
  6. To compile your game, run $t3make.
  7. Create a bash script named run.sh and give it the following code: {% include quickstart/shorter-run-script-code.md %} Then, do $chmod +x run.sh.
  8. {% include quickstart/happy-coding.html %}
  9. To test your game, execute $./run.sh.
